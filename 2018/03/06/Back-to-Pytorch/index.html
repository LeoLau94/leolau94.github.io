2018/3/10更新

发现一个不知道是PyTorch(0.30)还是CUDA(8.0)引起的一个bug，就是直接再代码中设置os.environ['CUDA_VISIBLE_DEVICES']，有时候会失效，也就是说，无论你设置为哪块GPU，它都只使用GPU0。暂时没发现引起这个bug的原因和出现的条件。因此，在命令行指定是最保险的做法。例如：CUDA_VISIBLE_DEVICES=1,2 python xx.py
